    # 配置git ssh问题

    问题如下：

    $ ssh -T git@github.com
/usr/bin/bash: line 0: exec: D:SetupMobaXtermMobaXterm.exe: not found
kex_exchange_identification: Connection closed by remote host
Connection closed by UNKNOWN port 65535

    遇到问题首先Google一下，有的解决方案认为超过了sshd的最大连接数，需要修改最大连接数并重启sshd服务。然而并没有什么用，应该和报错不相关。偶然看到知乎的一个解决方案，使用ssh -v + 需要连接的服务器进入debug模式查看是否详细的报错信息。

    $ ssh -v git@github
OpenSSH_8.5p1, OpenSSL 1.1.1k  25 Mar 2021
debug1: Reading configuration data /c/Users/xxxx/.ssh/config
debug1: /c/Users/xxxx/.ssh/config line 5: Applying options for *
debug1: Reading configuration data /etc/ssh/ssh_config
debug1: Executing proxy command: exec D:\\Setup\\MobaXterm\\MobaXterm.exe -H 127.0.0.1:10809 xx.xx.xx.xx 22
debug1: identity file /c/Users/xxxx/.ssh/id_rsa type 0
debug1: identity file /c/Users/xxxx/.ssh/id_rsa-cert type -1
debug1: identity file /c/Users/xxxx/.ssh/id_dsa type -1
debug1: identity file /c/Users/xxxx/.ssh/id_dsa-cert type -1
debug1: identity file /c/Users/xxxx/.ssh/id_ecdsa type -1
debug1: identity file /c/Users/xxxx/.ssh/id_ecdsa-cert type -1
debug1: identity file /c/Users/xxxx/.ssh/id_ecdsa_sk type -1
debug1: identity file /c/Users/xxxx/.ssh/id_ecdsa_sk-cert type -1
debug1: identity file /c/Users/xxxx/.ssh/id_ed25519 type 3
debug1: identity file /c/Users/xxxx/.ssh/id_ed25519-cert type -1
debug1: identity file /c/Users/xxxx/.ssh/id_ed25519_sk type -1
debug1: identity file /c/Users/xxxx/.ssh/id_ed25519_sk-cert type -1
debug1: identity file /c/Users/xxxx/.ssh/id_xmss type -1
debug1: identity file /c/Users/xxxx/.ssh/id_xmss-cert type -1
debug1: Local version string SSH-2.0-OpenSSH_8.5

    排查报错信息，确定是之前ssh连接服务器做了本地代理的配置，注释掉此注释即可。

    $ ssh -T git@github.com
Hi fightlee01! You've successfully authenticated, but GitHub does not provide shell access.
